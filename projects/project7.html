Multilingual Character Prediction Model

Python NLP
GitHub

Overview

Developed for CSE 447 (Natural Language Processing), this project created a lightweight predictive model for suggesting the top 3 most likely next characters in text input. The system uses a hybrid approach combining 5-gram and unigram character-level models to optimize for both accuracy and speed across multiple languages.

View Full Report

Technical Implementation

The core components of the prediction system included:

  • A hybrid 5-gram and unigram character-level model for prediction
  • Optimized data structures (defaultdict, Counter) chosen over external libraries to reduce runtime
  • Training on the OpenAssistant dataset with 161,433 messages across multiple languages
  • Preprocessing pipeline to filter non-UTF-8 characters while preserving punctuation, casing, and whitespace
  • Language-specific model adaptations to handle different character sets

Results

The final system achieved the following performance metrics:

  • 54.75% overall accuracy on the multilingual validation dataset
  • 62.94% accuracy on ASCII-based languages (340 samples)
  • 8.33% accuracy on non-Latin scripts (60 samples)
  • Strong performance on letter-based languages (English, Spanish, German, French, Vietnamese, Swedish)
  • Struggled with character-based languages (Chinese, Korean, Japanese) and non-Latin scripts (Russian, Hebrew, Arabic)

Future Improvements

While successful, the model could be enhanced through:

  • Better handling of non-Latin scripts, namely better training data
  • Implementation of more sophisticated backoff strategies for rare character sequences
